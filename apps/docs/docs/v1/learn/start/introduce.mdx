## 基本设计概念

本程序是基于 [***Electron***](https://www.electronjs.org/), [***React***](https://react.dev/) ，利用Web技术，以 [***Leafer***](https://www.leaferjs.com/) 为绘图引擎编写了一款桌面绘图应用程序。

用户可根据自身需求，在本程序得工作区建立绘图图稿进行创作，本程序将提供：图层、图形、自主绘制、图形绘制、属性设置、导入、导出、选择、线条、滤镜、标尺等工具，用户可以绘制自主图纸，也可以建立流程图等等。

旨编写一套快速迭代桌面应用程序的示例，编写项目快速启动编写的框架，本项目可供模板支持，开发基础和思路支持，但请勿直接使用。<span style={{ color: 'red' }}>如果需要直接使用本程序作为您得最终程序，请获得作者许可。</span>

你可以根据本程序, 进行快速迭代开发, 让你的生产力进行加速。

🌱 <span style={{ color: 'var(--rp-c-link)' }}>当前本程序正在开发当中, 还有着许多没有完善的地方, 如果你需要一个预览开发后结果程序的版本, 在 `pre-main-vue` 分支中含有一个使用 `vue` 编写的流程图绘制程序, 该版本是一个初期版本 Demo, 具有较多不合理的地方。</span>

### 项目组织

本项目当中采用了 `monorepo` 进行项目的编写. 利用 [***pnpm***](https://www.pnpm.cn/) & [***monorepo***](https://pnpm.io/zh/workspaces) 的方式组织项目

#### pnpm

* 快速：pnpm 通过链接共享依赖，减少安装时间和磁盘占用。

* 一致性：确保项目中依赖的版本一致，避免“依赖地狱”。

* 节省空间：使用硬链接和符号链接，多个项目共享相同的依赖。

* 兼容性：支持 npm 和 Yarn 的大部分功能，易于迁移。

* 工作区支持：方便管理多个包的单一代码库。

#### monorepo

* 集中管理：将多个产物（如桌面应用和网页应用）集中在一个仓库中，便于管理和协作。

* 版本一致性：可以确保所有相关产物使用相同的依赖版本，减少兼容性问题。

* 共享代码：不同产物之间可以方便地共享代码和组件，减少重复开发。

### 框架

对于本项目的框架，采用了 [***Electron***](https://www.electronjs.org/) 作为构建项目的基石。
采用 [***TypeScript超集***](https://www.typescriptlang.org/) 作为项目编写开发的技术语言，让项目获得严格的 TS 类型校验和提示。增强项目的可维护可迭代能力

#### electron

* 使用 HTML、CSS 和 JavaScript 开发应用，前端开发者易于上手。

* 支持 Windows、macOS 和 Linux，允许一次编写、处处运行。

* 可以访问 Node.js API，轻松处理文件系统和网络请求。

* 内置自动更新功能，简化用户的更新体验。

* 支持通知、系统托盘等桌面特性，提升用户体验。

* Electron 常用于构建像 Visual Studio Code、Slack 等流行应用。

#### typeScript

* TypeScript 是一种由微软开发的开源编程语言，基于 JavaScript 的超集。

* 它增加了静态类型检查，帮助开发者在编写代码时捕获潜在错误。

* 支持现代 JavaScript 特性，如 async/await、模块化等。

* 提供强大的工具支持，包括智能提示、重构和代码导航。

* 促进大型项目的可维护性和可读性，使团队协作更加高效。

* 可以与现有的 JavaScript 代码无缝集成，支持渐进式采用。

* TypeScript 编译后生成标准 JavaScript，确保广泛的兼容性。

* 许多流行框架（如 Angular、React）都支持 TypeScript，提升了开发体验。

### UI

在处理应用程序的图形界面时，采用 [***React***](https://react.dev/) 作为开发页面的主导框架、
以 [***Leafer***](https://www.leaferjs.com/) 为绘图引擎，同步协调工作，进而使得绘图体验以及程序界面体验都能获得良好的感受，

#### react

* 组件化设计，允许开发者创建可重用的 UI 组件。

* 使用虚拟 DOM 提升性能，优化更新和渲染效率。

* 单向数据流，便于管理和调试应用状态。

* 支持声明式编程，使 UI 开发更加直观和可预测。

* 拥有丰富的生态系统，提供众多库和工具以增强功能。

* 易于与其他库和框架集成，灵活性强。

* 广泛应用于单页应用和移动应用，提升用户体验。

#### leafer、 leafer-ui

* Leafer UI 是基于 Leafer 开发的一套绚丽多彩的 UI 绘图、交互框架，可结合 AI 绘图、生成界面，表现力丰富、性能强劲。

* 提供了丰富的 UI 绘图元素，和开箱即用的功能，如自动布局、图形编辑、SVG 导出，方便与 Figma、Sketch 等产品进行数据交换。

* 可用于高效绘图 、UI 交互（小游戏、互动应用、组态）、图形编辑。

### 存储

应用程序的数据存储往往需要为空间考虑，为安全性考虑。在空间上，采用 [***Pako***](https://nodeca.github.io/pako/) 作为压缩数据的工具，再利用 `@suey/pkg-utils` 封装的数据加密解密算法进行一般性加密。
最终利用 `nodejs` 中 `fs` 模型进行读写文件、又或者利用 [***ElectronStore***](https://www.npmjs.com/package/electron-store) 作为存储媒介存储数据。

#### electron-store

* 提供简单的 API，方便开发者快速集成。

* 支持自动数据持久化，应用重启后数据仍然可用。

* 支持跨平台，适用于 Windows、macOS 和 Linux。

* 具有良好的性能，适合存储用户设置和应用状态。

#### pako.js

* 支持多种压缩格式，如 DEFLATE、GZIP 和 ZLIB。

* 提供简单的 API，易于集成到 Web 应用和 Node.js 项目中。

* 在浏览器和服务器端均可运行，兼容性强。

* 具有较小的体积，适合用于资源受限的环境。

* 高性能，能够快速处理大数据集，提升应用效率。

* 支持流式处理，适合需要实时数据压缩的场景。

* 社区活跃，提供丰富的文档和示例，便于开发者使用。

### 测试

#### jest

* 快速：内置并行测试运行，提升测试速度。

* 丰富的匹配器：提供多种匹配器，使断言更加灵活和易读。

* 支持快照测试：可以快速捕捉组件输出并进行对比。

* 强大的模拟功能：方便模拟函数和模块，简化依赖管理。

* 易于集成：与其他工具和框架（如 Babel 和 TypeScript）兼容良好。
